## 1. 架构全景与核心理念

### 1.1 系统定位
“弈”不再仅仅是一个聊天界面，而是一个 **“AI 工作单元调度中心”**。用户、Agent、系统定时器三者平等地向“事件中心（Event Hub）”投递“事件（Event）”，调度器负责消费并分配角色执行。

### 1.2 三大设计原则
1. **统一接入（Unified Ingress）**：无论是用户手动点击、Agent 顿悟触发，还是凌晨定时任务，都归一化为标准的 `Event` 记录。
2. **执行可视化（Visual Execution）**：所有事件在执行时，都会生成一个 **“只读会话（Read-only Session）”**，用户可在聊天界面实时观看 AI 的每一步操作。
3. **来源可溯（Source Traceability）**：每个事件都携带 `source_type`（user/agent/system），实现全链路审计。

## 2. 数据模型定义（Schema DDL）

### 2.1 事件表（`events`）—— 核心中枢
```sql
CREATE TABLE events (
  id TEXT PRIMARY KEY,
  
  -- 来源系统
  source_type TEXT NOT NULL CHECK(source_type IN ('user', 'agent', 'system')),
  source_id TEXT,                 -- user_id / agent_id / null (system)
  source_meta TEXT,               -- JSON: { session_id, trigger, ui_action }
  
  -- 执行指派
  assigned_agent_id TEXT NOT NULL,-- 如 'sys_evolver'
  assigned_group_id TEXT,
  
  -- 定义
  type TEXT NOT NULL CHECK(type IN ('once', 'cron')),
  cron_expr TEXT,                 -- 仅 cron
  payload TEXT NOT NULL,          -- JSON: { instruction, context, ttl }
  
  -- 状态机
  status TEXT DEFAULT 'pending' CHECK(status IN ('pending', 'paused', 'running', 'success', 'failed', 'expired')),
  priority INTEGER DEFAULT 0,
  
  -- 时间审计
  scheduled_at DATETIME,
  started_at DATETIME,
  finished_at DATETIME,
  result_summary TEXT,
  error_log TEXT,
  
  -- 链式追踪
  parent_event_id TEXT,
  retry_count INTEGER DEFAULT 0,
  max_retries INTEGER DEFAULT 3,
  
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_events_status_schedule ON events(status, scheduled_at);
CREATE INDEX idx_events_source ON events(source_type, source_id);
```

## 4. 核心流程实现规范（按执行顺序）

### 4.1 流程一：Session 完成 -> 投递事件
**文件**：`apps/server/src/agent/outer.ts:499-515`
**触发时机**：Session 结束时（task_complete / max_turns / stop / cancelled）。

```typescript
// 实际代码：dispatchSessionCompletedEvent (outer.ts)
function dispatchSessionCompletedEvent(session: SessionRow) {
  if (session.session_type === 'event') return        // 事件会话不再投递事件
  try {
    eventService.create({
      source_type: 'agent',
      source_id: session.character_id,
      source_meta: { session_id: session.id, trigger: 'session.completed' },
      assigned_agent_id: 'sys_evolver',
      type: 'once',
      payload: { instruction: `Session ${session.id} completed, analyze trajectory for potential skill extraction` },
      status: 'paused',                                // 固定 paused，等待审批
      scheduled_at: Date.now(),
    })
  } catch (err) {
    console.warn('[session.completed] Failed to dispatch event:', err)
  }
}
```

### 4.3 流程三：事件调度器消费（核心枢纽）
**文件**：`apps/server/src/event/eventScheduler.ts`（调度器），`apps/server/src/event/eventExecutor.ts`（执行器）
**运行机制**：每 10 秒轮询一次 `pending` 事件。

```typescript
// 实际代码：调度器 (eventScheduler.ts)
async function poll(io: Server) {
  if (isPolling) return
  isPolling = true
  try {
    const events = eventService.getPending(MAX_CONCURRENT)  // SELECT status='pending' AND scheduled_at <= now()
    for (const evt of events) {
      eventService.updateStatus(evt.id, 'running')
      io.emit('event:status_changed', { eventId: evt.id, status: 'running' })
      executeEvent(evt, io).catch(...)
    }
  } finally {
    isPolling = false
  }
}

// 实际代码：执行器 (eventExecutor.ts)
export async function executeEvent(evt: EventRow, io: Server): Promise<void> {
  const payload = JSON.parse(evt.payload || '{}')
  const { instruction, context } = payload
  const agentId = evt.assigned_agent_id

  // 1. 创建只读会话
  const sessionId = `evt_${evt.id}_${Date.now()}`
  sessionStore.create({
    id: sessionId,
    character_id: agentId,
    title: `📋 ${instruction.slice(0, 50)}`,
    session_type: 'event',
    event_id: evt.id,
    is_readonly: 1,
  })

  // 2. 注入指令并通知前端
  messageStore.addMessage(sessionId, { role: 'user', content: instruction })
  if (context) messageStore.addMessage(sessionId, { role: 'user', content: `[Context]\n${JSON.stringify(context)}` })
  io.emit('session:new', { sessionId, title: instruction.slice(0, 50), isEvent: true })

  // 3. 复用 sessionLoop（outer.ts）
  const result = await sessionLoop(io, makeFakeSocket(io), sessionId)

  // 4. 收尾
  const status = result.status === 'cancelled' ? 'failed' : 'success'
  const msgs = messageStore.getMessages(sessionId, 1)
  const resultSummary = msgs[msgs.length - 1]?.content?.slice(0, 500)
  await eventService.updateStatus(evt.id, status, { result_summary: resultSummary })
  io.emit('event:status_changed', { eventId: evt.id, status, result_summary: resultSummary })
}
```

function checkStrategy(toolName: string, strategy: Strategy): 'allow' | 'ask' | 'deny' {

  if (strategy === 'Ask' && dangerous) return 'ask'
