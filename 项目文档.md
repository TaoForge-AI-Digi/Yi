## 一、项目概述

弈是一个 AI Agent 对话与自动化平台。用户可与配置了独立人格（Soul）、工具集和技能的 Agent 对话，Agent 可执行代码/文件操作、调用 MCP 服务、委托子任务，并能基于运行轨迹自动沉淀技能。

### 核心理念

- **有灵魂** — 每个 Agent 有独立人格（SOUL.md）、记忆（MEMORY.md）、用户画像（USER.md）
- **有策略** — Plan（只读分析）/ Ask（执行前确认）/ Bypass（自主执行）三种模式手动切换
- **有分身** — 主 Agent 可委托 Sub-agent 并行处理子任务，结果自动回收
- **有工具** — Agent 携带 read/write/edit/bash/grep/glob/webfetch/websearch 等工具
- **会进化** — 在线检测洞察 + 离线聚类沉淀 Skill

## 四、核心架构

### 4.1 总体架构

```
用户 (Web UI)
    │
    ├── HTTP REST ──→ Hono Server ──→ SQLite
    │                    │
    └── Socket.IO ──────┤
                        │
                    ┌───┴───┐
                    │ Agent  │
                    │ Engine │
                    └───┬───┘
                        │
                    ┌───┴───┐
                    │  LLM   │
                    └───────┘
```

### 4.2 Agent 双循环引擎

**外层循环 (Session Loop)** — `apps/server/src/agent/outer.ts`

1. 验证会话和角色存在性
2. 加载角色身份：Soul + User + Memory
3. 解析工具绑定（含 MCP 自动连接，指数退避重试）
4. 构建 System Prompt（含 guidance 块、工具列表、技能索引）
5. 加载消息历史，展开 `@file/@folder/@url` 引用
6. 进入主循环（最多 20 轮）：
   - 调用内层循环 `innerLoop`
   - 处理 Sub-agent 信号（拉起 → 回收 → 注入）
   - 处理 `task_complete` 信号
   - Doom Loop 检测（连续 6 次失败或重复模式）
   - Token 压缩（超过 context_window 的 75% 时执行）
7. 会话结束时检测进化洞察，投递事件

**内层循环 (Processor Loop)** — `apps/server/src/agent/inner.ts`

1. 调用 LLM（支持流式、重试 3 次、指数退避）
2. 解析 LLM 响应：
   - 无工具调用 → 作为 final_answer 返回
   - `delegate_task` → 返回 sub_agent_request 信号给外层
   - `task_complete` → 返回 task_complete 信号给外层
3. 工具调用执行（分 4 阶段）：
   - Phase 1: 预检所有工具（L1 绑定校验 + L2 策略拦截 + Ask 模式用户确认）
   - Phase 2: 发送 tool.started 事件
   - Phase 3: 发送被拒绝工具的错误结果
   - Phase 4: 只读工具并行执行，写入工具串行执行

### 4.3 权限模型

| 阶段 | 引擎 | 检查内容 |
|------|------|---------|
| L1 身份绑定 | 内层循环 | 工具是否在角色的 `tools` 白名单中？约束条件（路径/命令）是否满足？ |
| L2 策略拦截 | 内层循环 | 当前策略是否允许？Plan 拒绝危险工具，Ask 需用户确认，Bypass 放行 |

危险工具列表（硬编码）：`write`, `edit`, `delete`, `bash`, `mcp_exec`, `delegate_task`, `skill_manager`, `character_manager`

### 4.4 工具系统（插件式）

每个工具是一个独立目录，包含 `tool.json`（元数据 + 约束字段定义）和 `index.ts`（实现）。

**内置工具：**

| 工具 | 说明 | 危险 |
|------|------|:----:|
| `read` | 读取文件内容 | 否 |
| `write` | 写入文件 | 是 |
| `edit` | 精确替换编辑 | 是 |
| `grep` | 正则内容搜索 | 否 |
| `glob` | 文件模式匹配 | 否 |
| `bash` | 执行 shell 命令 | 是 |
| `webfetch` | 获取 URL 内容 | 否 |
| `websearch` | 联网搜索 | 否 |
| `skill_manager` | 技能管理（view/create/edit/delete） | 是 |
| `character_manager` | 角色管理 | 是 |
| `debug_sessions` | 调试会话管理 | 是 |
| `delegate_task` | 委托子任务（signal 工具） | 是 |
| `task_complete` | 标记任务完成（signal 工具） | 否 |

### 4.5 Sub-agent 机制

- `delegate_task` 工具拉起同组 Sub-agent
- 组校验 + 角色校验（仅 sub/both 角色可被委托）
- 子会话独立上下文（隔离执行）
- 最大递归深度 3 层（depth≥1 的 Sub-agent 会移除 delegate_task 以防止无限递归）
- 结果压缩回收（`summarizeAndMerge`）

### 4.6 事件引擎

- **事件表**（`events`）：存储所有事件，含来源类型（user/agent/system）、状态、优先级、Cron 表达式
- **轮询调度器**（`eventScheduler.ts`）：每 10 秒扫描 pending 事件
- **事件执行器**（`eventExecutor.ts`）：创建只读会话 → 复用 sessionLoop
- **Cron 续期**：`completeAndRequeue` 自动为 cron 事件创建下一次
- **事件类型**：`once`（一次性）/ `cron`（定时）
- **事件状态**：`pending → running → completed / failed`

### 4.7 进化引擎

**在线检测**（`onlineDetector.ts`）：会话结束时检测
- 高错误率
- 重复工具模式
- 高频工具调用
- 检测到洞察后投递进化事件

**离线聚类**（`offlineMiner.ts`）：每日凌晨 2:00 定时触发
- LCS（最长公共子序列）计算轨迹相似度
- Union-Find 连通分量聚类
- InsightExtractor 从聚类中提取技能草稿
- SkillGenerator 生成 SKILL.md 文件

## 七、数据流

### 7.1 用户对话流程

```
用户输入 → socket 'chat-run'
  → sessionLoop(outer.ts)
    → 验证会话/角色/供应商/模型
    → 加载角色身份 (Soul + User + Memory)
    → MCP 自动连接（失败重试 3 次，指数退避）
    → 构建 System Prompt（guidance + 工具列表 + 技能索引）
    → 加载消息历史 & 展开引用
    → innerLoop (最多 20 轮)
      → LLM 流式调用（重试 3 次）
      → 解析响应
        → delegate_task → 外层拉起 Sub-agent → 回收结果
        → task_complete → 终止
        → 工具调用 → L1 校验 → L2 校验 → 执行 → 追加结果
      → Doom Loop 检测 → Token 压缩
    → 会话结束 → 进化洞察检测 → 投递事件
```

### 7.2 事件执行流程

```
事件创建（用户/Agent/系统）→ events 表 status=pending
  → EventScheduler 轮询（每 10s）
    → status → running
    → EventExecutor:
      → 创建只读会话（session_type=event, is_readonly=true）
      → 通知前端 session:new
      → sessionLoop（强制 Bypass 策略）
      → 事件完成 → status → completed
      → Cron 事件自动续期
```

### 7.3 进化流程

```
会话结束 → Outer 检测工具调用记录
  → detectInsight (onlineDetector)
    → 高错误率 / 重复模式 / 高频调用
  → 投递进化事件（assigned_agent_id=evolver）
  → EventScheduler 消费 → Evolver 执行
    → 生成 SKILL.md

每日 2:00 → Cron 触发离线复盘事件
  → Evolver 读取近 7 天轨迹
  → OfflineMiner.mine (LCS + Union-Find)
  → InsightExtractor.extract (聚类 → 草稿)
  → SkillGenerator.generate (草稿 → SKILL.md)
```
