## 一、项目概述

弈 (Yi-Lin) 是一个 Coding Agent 聊天应用。架构上采用 **Server 运行完整 Agent Loop，Client 只做展示和交互** 的分离模式。

- **前端**: Vue 3 + Vite + TypeScript + Socket.IO Client + Pinia + vue-i18n
- **后端**: Node.js + Hono + Socket.IO + better-sqlite3
- **存储**: SQLite (sessions/messages) + JSON (providers/characters metadata) + MD 文件 (character content)

---

## 二、设计阶段（早期）

### 2.1 设计总纲

`docs/arch/design.md` 定义了整体架构：
- **架构图**: Vue 3 Client ↔ Socket.IO ↔ Hono Server (Agent Loop)
- **数据模型**: sessions/messages 表 (SQLite)、providers (JSON)、characters (分片存储：元数据 JSON + soul.md/user.md/memory.md)
- **组件树**: ChatPage → Sidebar/ChatArea/SettingsModal/RoleSettings
- **Agent Loop**: chat-run → 构建 System Prompt → 加载历史 → callLLM(stream) → parseToolCalls → executeTool → 循环
- **工具定义**: read/write/edit/bash/grep/glob/webfetch/websearch
- **权限模型**: CharacterPermission (allow/ask/deny)
- **Socket.IO 协议**: 定义完整的 C→S / S→C 事件
- **HTTP API**: RESTful CRUD for providers/sessions/characters

### 2.2 竞品对比

`docs/arch/comparison-with-hermes.md` 和 `three-way-comparison.md` 做了两次对比分析：

- 与 Hermes Agent (opencode) 对比：81 文件 vs 6924 文件
- Yi-Lin 缺失的关键工具：task_complete、memory_tool、patch_file、skill_manage、session_search、browser、git 系列
- 结论：优先补齐 task_complete + memory_tool + patch_file，架构级跟进工具注册中心 + 技能系统 + 上下文压缩

## 四、已完成的开发（按 Phase 顺序）

### Phase 0：术语对齐与基础设施

- `mode` → `role` 统一 (main/sub/both)
- `CharacterRecord` 补字段：groups、default_strategy、skills
- 角色独立目录存储：`data/characters/{id}/character.json` + soul.md/user.md/memory.md
- `RoleSettings.vue` skills tab 占位符填实——技能多选器
- 全局 agent → character 术语替换

### Phase 1：权限模型重构

- SessionState + 策略命令 (Plan/Ask/Bypass)
- WebSocket 命令 `strategy.set` / `strategy.updated`
- `DANGEROUS_TOOLS` 硬编码列表（write/edit/delete/bash/mcp_exec/delegate_task 等）
- L2 策略拦截：Plan 拒绝危险工具，Ask 弹窗确认，Bypass 放行
- 前端 `ApprovalDialog.vue` 策略适配
- `StrategyToggle.vue` + `StrategyIndicator.vue`

### Phase 2：调度引擎重构

- `outer.ts` / `inner.ts` 拆分（原 `loop.ts` 230 行一分为二）
- 外层循环 (sessionLoop)：身份加载 → 会话状态 → 主循环
- 内层循环 (innerLoop)：LLM 推理 → Tool Call 检测 → 两阶段校验 → 执行
- Doom Loop 检测：连续 6 次失败或重复模式时发送系统警报
- 上下文压缩检测

### Phase 3：Sub-agent 机制

- `delegate_task` 工具定义（task/target_character_id/sub_strategy/instances）
- 组校验 + 角色校验（同组约束，sub/both 角色校验）
- `spawnAndRunSubAgent()`：创建子会话 → 递归深度限制 (MAX_DEPTH=3) → depth≥1 移除 delegate_task
- `summarizeAndMerge()`：多实例结果压缩合并

### Phase 4：能力引擎完善

- **4.1 Skill Index**: `buildSkillIndex()` 按角色 skills 白名单过滤，注入 system prompt 的 Level 0 索引
- **4.2 skill_manager**: view/create/edit/delete/write_file/remove_file 全链路（list 已移除——白名单已在 system prompt 中离线注入）
- **4.3 task_complete**: Agent 主动结束信号 + 输出摘要
- **4.4 Skill 绑定 UI**: RoleSettings 技能多选器
- **4.5 MCP 执行**: 计划制定但跳过（见下文超能力计划）

### 前端已完成功能

- MCP server 管理页面：ToolSettings.vue
