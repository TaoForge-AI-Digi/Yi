## Phase 5：进化引擎

> **设计约束：** agent 自主创建/修改 skill 仅限于 `skills/user/` 目录，系统预设 skill（`skills/{category}/`）只读。

### 5.1 前置条件 ✅ 已满足

- `skill_manager` 的 `create` / `edit` / `delete` / `write_file` / `remove_file` 已就绪
- `characterMetaStore.update()` 可用于绑定 skill 到角色
- `outer.ts` 的 `sessionLoop` 中 `toolCallHistory`、`messages`、`turn`、`session` 等轨迹数据均可获取

### 5.2 新建文件：`apps/server/src/agent/evolution.ts`

**类型定义：**

```typescript
interface Trajectory {
  toolCallCount: number
  success: boolean
  hasSelfCorrection: boolean
  hasUserCorrection: boolean
  finalGoal: string
  initialQuery: string
  finalAnswer: string
  toolCalls: string[]
  errors: { toolName: string; error: string }[]
  criticalActions: string[]
}

interface Insight {
  title: string
  content: {
    problem: string
    solution: string
    keySteps: string[]
    pitfalls: string[]
  }
  tags: string[]
}
```

**核心函数：**

| 函数 | 行数 | 说明 |
|------|------|------|
| `buildTrajectory()` | ~30 | 从 outer.ts 传递的运行时数据组装 Trajectory |
| `shouldTriggerEpiphany()` | ~10 | ≥3 次 tool call + 成功，或检测到自我纠正 |
| `extractInsight()` | ~20 | 提取初始问题、解决方案、关键步骤、错误 |
| `renderSkillTemplate()` | ~20 | 生成含 frontmatter 的 SKILL.md |
| `generateSkill()` | ~25 | 写入 `skills/user/{slug}/SKILL.md` + 绑定到角色 |
| `runEvolution()` | ~15 | 统一入口：组装轨迹 → 检测 → 提取 → 生成 |

**顿悟触发条件（`shouldTriggerEpiphany`）：**

```
toolCallCount >= 3 && success → true
hasSelfCorrection              → true
hasUserCorrection              → true
否则                           → false
```

**数据来源映射：**

| Trajectory 字段 | outer.ts 中来源 |
|---|---|
| `toolCallCount` | `toolCallHistory.length` |
| `success` | loop 正常结束（非 cancelled / max_turns）|
| `hasSelfCorrection` | toolCallHistory 中 error 后跟了不同 tool |
| `hasUserCorrection` | messages 中 user 行出现在 assistant/tool 之后 |
| `finalGoal` | `session.title` 或第一条 user message |
| `initialQuery` | messages 中第一个 user 消息 |
| `finalAnswer` | messages 中最后一个 assistant 消息 content |
| `toolCalls` | toolCallHistory 中的 toolName（去重）|
| `errors` | toolCallHistory 中 hasError 的记录 |
| `criticalActions` | 过滤掉 read-only 后的 tool 列表 |

### 5.3 修改文件：`apps/server/src/agent/outer.ts`

在 `sessionLoop` 的 loop 结束后（token 保存段），对成功完成的会话调用 `runEvolution()`：

```
completedStatus === 'stop'       → 调用
completedStatus === 'task_complete' → 调用
cancelled / max_turns            → 跳过
```

注意 `task_complete` handler 有独立的提前 return 路径，需同时覆盖。

### 5.4 验收标准

- 成功会话（≥3 次 tool call，含错误后纠正）自动在 `skills/user/{slug}/` 生成 SKILL.md
- 自动生成的 skill 自动绑定到当前角色 `skills[]` 列表
- 角色在下一次会话中可在 `## Available Skills` 中看到自动生成的 skill
- 失败/取消/超限的会话不触发生成
- `styles/user/` 下的 skill 不影响系统 skill
